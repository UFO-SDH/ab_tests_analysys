# A/B Test Analysis — примеры статистического анализа экспериментов

Репозиторий содержит практические примеры полного цикла анализа A/B и A/B/C тестов: от валидации данных и проверки SRM до оценки статистической значимости и построения доверительных интервалов.

## Кейсы

### 1. A/B тест — анализ ARPU (игровое приложение)

- **Метрика:** ARPU (Average Revenue Per User)
- **Обработка данных:**
  - Проверка трафика на пропуски (дни без пользователей, дни без конверсий)
  - Очистка данных: удаление пользователей с офисным IP и попавших в обе группы
  - Проверка SRM (Sample Ratio Mismatch)
  - Проверка однородности выборок по браузерам (Chi-squared)
  - t-test (Welch's) для ARPU
  - Проверка побочной метрики (load time)
- **Результат:** тестовый вариант статзначимо лучше (p = 0.0096 < α = 0.01)

### 2. A/B/C тест — сравнение трёх вариантов по ARPU

- **Метрика:** ARPU
- **Обработка данных:**
  - SRM-проверка баланса трёх групп
  - Попарное сравнение вариантов (t-test from stats)
  - Поправка на множественные сравнения (Bonferroni: α = 0.05 / 2 = 0.025)
- **Результат:** вариант B статзначимо лучше A; вариант C не показал значимого отличия

### 3. A/B тест — время обработки счёта (сервис для бухгалтеров)

- **Метрика:** среднее время обработки счёта (мин.)
- **Обработка данных:**
  - Очистка выбросов (> 120 мин.)
  - Визуализация динамики метрики по дням — выявлен эффект первичности (novelty effect)
  - Агрегация на уровне пользователя
  - SRM-проверка (Chi-squared)
  - t-test (Welch's)
  - Построение 95% доверительного интервала
- **Результат:** тестовый вариант снизил время обработки на 2.57–2.99 мин. (p ≈ 0, CI 95%)
